Sistem Bilangan dan Konversi dalam Dunia Komputer

Dalam dunia teknik komputer dan jaringan, pemahaman sistem bilangan merupakan dasar penting. Sistem bilangan yang paling sering digunakan adalah biner, desimal, oktal, dan heksadesimal.

Bilangan biner hanya terdiri dari angka 0 dan 1. Sistem ini digunakan oleh mesin komputer karena mencerminkan sinyal digital (ON/OFF).

Bilangan desimal adalah sistem angka yang umum digunakan manusia, terdiri dari angka 0 sampai 9.

Bilangan oktal (basis 8) dan heksadesimal (basis 16) biasa digunakan untuk menyederhanakan tampilan angka biner panjang, terutama dalam penulisan alamat memori atau konfigurasi sistem.

Peserta PKL dilatih melakukan konversi antar sistem bilangan, seperti:

Konversi dari desimal ke biner dilakukan dengan membagi angka desimal dengan 2 secara berulang.

Konversi dari biner ke desimal dilakukan dengan menjumlahkan hasil perkalian setiap bit biner dengan pangkat dua sesuai posisinya.

Konversi juga dilakukan ke dan dari sistem oktal dan heksadesimal.

Materi ini menjadi landasan dalam memahami alamat IP, subnet mask, dan konfigurasi lainnya dalam jaringan.

IP Address, Kelas IP, dan Teknik Subnetting

Peserta PKL mempelajari secara mendalam tentang pengalamatan IP dan teknik subnetting.

IP Address adalah alamat logis yang digunakan setiap perangkat dalam jaringan untuk saling berkomunikasi. Materi ini mencakup:

Struktur IP versi 4 (IPv4) yang terdiri dari empat oktet (contoh: 192.168.1.1).

Pembagian kelas IP berdasarkan range:

Kelas A: 1.0.0.0 – 126.255.255.255

Kelas B: 128.0.0.0 – 191.255.255.255

Kelas C: 192.0.0.0 – 223.255.255.255

Selain itu, peserta juga mempelajari Subnetting, yaitu membagi satu jaringan besar menjadi jaringan kecil agar pengalokasian IP lebih efisien dan tertata.

Praktik yang dilakukan termasuk:

Menentukan jumlah host dan subnet dengan rumus 2ⁿ - 2.

Menentukan network ID, broadcast address, IP awal, dan IP akhir.

Menggunakan prefix subnet, seperti /25, /26 hingga /30, untuk membagi jaringan menjadi kelompok lebih kecil yang disesuaikan kebutuhan jaringan.

Materi ini sangat penting karena hampir semua sistem jaringan mengandalkan pengaturan IP dan subnetting yang benar agar komunikasi data berjalan optimal.

Instalasi Sistem Operasi dan Konfigurasi Server

Pada tahapan ini, peserta menginstal dan mengelola berbagai server menggunakan sistem operasi Linux Debian 11.5. Materi ini bersifat praktik dan membutuhkan ketelitian. Beberapa server yang dikonfigurasi antara lain:

DNS Server: Berfungsi untuk menerjemahkan nama domain ke alamat IP. Peserta mengkonfigurasi dua domain lokal sebagai latihan.

Web Server: Digunakan untuk menampilkan halaman web. Peserta menggunakan Joomla, WordPress, dan Opencart untuk membangun halaman web secara mandiri.

FTP Server: Untuk transfer file antar komputer dalam jaringan.

Email Server: Untuk mengatur pengiriman dan penerimaan email secara lokal.

Proxy Server: Menjadi perantara antara pengguna dan internet, juga menyimpan cache untuk efisiensi akses.

File Server: Berfungsi untuk menyimpan dan membagikan file ke seluruh pengguna jaringan.

Monitoring Server (Munin): Untuk memantau status server dan jaringan.

DHCP Server: Memberikan IP secara otomatis kepada perangkat yang terhubung ke jaringan.

Peserta juga diajarkan perintah-perintah dasar Linux dan penggunaan script konfigurasi. Semua kegiatan dilakukan mulai dari konfigurasi manual file /etc/network/interfaces, perintah terminal, hingga menjalankan dan mengaktifkan layanan dengan systemctl.

Mikrotik, Firewall, dan Konfigurasi Access Point

Bagian akhir dari kegiatan PKL adalah mempelajari penggunaan router Mikrotik, konfigurasi firewall, dan pengaturan access point. Mikrotik adalah alat penting dalam pengaturan jaringan, khususnya pada jaringan menengah hingga besar.

Peserta mempelajari:

Konfigurasi dasar Mikrotik menggunakan aplikasi Winbox dan command line interface (CLI).

Firewall Mikrotik: Membuat aturan keamanan untuk memfilter dan mengatur lalu lintas data.

Hotspot: Membuat sistem login pengguna sebelum dapat mengakses internet.

Network Address Translation (NAT): Menyambungkan jaringan lokal ke jaringan publik (internet).

Selain itu, peserta juga melakukan:

Pemasangan Access Point (AP) baik dalam mode router maupun bridge.

Konfigurasi SSID, keamanan jaringan (password), dan channel Wi-Fi.

Pengujian jaringan nirkabel dengan menghubungkan perangkat client ke AP dan mengecek kestabilan koneksi.
